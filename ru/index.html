Существующие раличия описанны на странице Bad section id: differences-from-the-original-implementation.

cl-closure-template реализует систему шаблонов для серверной и клиенсткой стороны, котороая позволяет динамически создавать повторно используемые HTML и UI компоненты. Язык шаблонов имеет простой, естественный для программистов, систаксис. В отличие от традиционных систем, в которых каждый шаблон описывает целую страницу, вы можете использовать cl-closure-template для создания небольших компонентов и комбинировать их для создания пользовательского интерфейса.

Преимущества использования cl-closure-template¶

  • Удобство. cl-closure-template обеспечивает простой способ создания фрагментов разметки и отделения логики приложения от представления.

  • Язык шаблонов не зависит от используемого языка программирования и реализован для JavaScript и Common Lisp. Вы можете написать один шаблон и использовать его как на клиенте, так и на сервере.

  • Легкость чтения. Вы можете чётко видеть структуру генерируемого HTML глядя на стуктуру кода шаблонов.

  • Язык шаблонов ориентирован на программистов. Шаблоны это просто функции, которые могу вызывать друг друга. Синтаксис включает в себя конструкции, знакомые любому программисту. В однин исходный файл можно поместить несколько шаблонов.

  • Инструмент, а не фреймворк. cl-closure-template может быть использованна вместе с любой библиотекой или фреймворком.
